## Project Overview

AI Nexus Link is a modular cognitive automation platform with three distinct layers:
- **Frontend** (`/frontend`): Next.js app displaying marketing pages and blog posts pulled from WordPress
- **Orchestrator** (`/orchestrator`): Python agents that generate AI-written content and publish it to WordPress
- **Data layer**: Directus headless CMS as the primary data store (collections: `agent_config`, `agent_logs`, `client_leads`)

The WordPress instance at `https://www.digitalnapomoc.sk` serves as the headless content backend. Content is generated by the Z.ai GLM model (Chinese LLM API) and posted there as drafts.

## Commands

### Frontend (Next.js)
```bash
cd frontend
npm run dev      # dev server
npm run build    # production build
npm run lint     # ESLint
```

### Orchestrator (Python)
```bash
cd orchestrator
source venv/bin/activate
python main.py               # test Directus connection
python wp_writer_agent.py    # generate + post article to WordPress
```

## Architecture

### Frontend
- Uses Next.js App Router (`app/` directory) with React Server Components
- `lib/wp.ts` — WordPress REST API client; runs server-side only. Fetches posts with 5-minute ISR cache (`next: { revalidate: 300 }`). Returns empty array on error so blog section degrades gracefully.
- UI is dark-themed with glassmorphic design; accent color `#ff6b35`; Slovak locale throughout (dates formatted as "15. októbra 2024")

**Important:** The `frontend/AGENTS.md` warns that this Next.js version has breaking changes from what you may know — check `node_modules/next/dist/docs/` before writing Next.js-specific code.

### Orchestrator
- `wp_writer_agent.py`: Two-phase pipeline — (1) calls Z.ai `/v4/chat/completions` to generate a 1000+ word Slovak HTML article, (2) POSTs it to WordPress REST API as a draft using Basic Auth with an application password.
- `main.py`: Validates Directus connectivity using a Bearer token.
- All credentials come from `orchestrator/.env` — never hardcode them.

### Environment Variables
| Variable | Where used | Purpose |
|---|---|---|
| `WP_URL` | frontend `.env.local`, orchestrator `.env` | WordPress base URL |
| `WP_USER` | orchestrator `.env` | WordPress username |
| `WP_APP_PASSWORD` | orchestrator `.env` | WordPress application password |
| `ZAI_API_KEY` | orchestrator `.env` | Z.ai API key for GLM model |
| `DIRECTUS_URL` / `DIRECTUS_TOKEN` | orchestrator `.env` | Directus CMS access |

## Conventions

- Language of content and user-facing strings: **Slovak**
- WordPress REST API must not be public-facing — access is via application passwords only
- Orchestrator uses Python with `requests` and `python-dotenv`; no framework
